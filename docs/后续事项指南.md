## 第 1 件：本地启动 MySQL + Redis（开发调试用）

> 如果只打算用云托管、不需要本地调试，可以跳过这步。

### 操作步骤

1. 启动 MySQL 和 Redis（Docker 方式推荐）：

   ```bash
   # MySQL
   docker run -d --name mysql8 -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_DATABASE=arguing_tutorial mysql:8

   # Redis
   docker run -d --name redis -p 6379:6379 redis:7
   ```

2. 初始化数据库：

   ```bash
   mysql -u root -p123456 arguing_tutorial < arguing-tutorial-server/src/main/resources/db/migration/V1__init_schema.sql
   ```

3. 启动后端：

   ```bash
   cd arguing-tutorial-server
   set JAVA_HOME=D:/software/Java/jdk-17
   mvn spring-boot:run
   ```

4. 验证：浏览器访问 `http://localhost:8080/api/scenes` 应该能看到 5 条场景数据

> **注意**：本地开发时 server.port 需改回 8080（当前配置为 80，是给云托管用的）。

## 第 2 件：配置外部服务密钥

当前代码已接入智谱 GLM、阿里云语音、腾讯云 COS，需要填入真实密钥才能运行。

### 操作步骤

编辑 `arguing-tutorial-server/src/main/resources/application.yml`，或通过环境变量覆盖：

| 配置项 | 环境变量 | 获取方式 |
|--------|---------|---------|
| AI API Key | `AI_API_KEY` | 智谱开放平台 https://open.bigmodel.cn/ → API Keys |
| 阿里云 AccessKey ID | `ALI_ACCESS_KEY_ID` | 阿里云控制台 → AccessKey 管理 |
| 阿里云 AccessKey Secret | `ALI_ACCESS_KEY_SECRET` | 同上 |
| 阿里云语音 AppKey | `ALI_APP_KEY` | 阿里云「智能语音交互」→ 项目管理 → AppKey |
| 腾讯云 COS SecretId | `COS_SECRET_ID` | 腾讯云控制台 → 访问管理 → API 密钥管理 https://console.cloud.tencent.com/cam/capi |
| 腾讯云 COS SecretKey | `COS_SECRET_KEY` | 同上 |

### 本地开发快速配置（直接改 yml）

```yaml
ai:
  api-key: 你的智谱API Key

ali:
  access-key-id: 你的阿里云AK
  access-key-secret: 你的阿里云SK
  app-key: 你的语音AppKey

storage:
  cos:
    secret-id: 你的腾讯云SecretId
    secret-key: 你的腾讯云SecretKey
```

### 验证

启动后端后，用 curl 测试 AI 是否正常回复：

```bash
# 获取游客 token
curl -X POST http://localhost:8080/api/auth/guest

# 用返回的 userId 和 guestToken 测试创建会话（需要真实场景 ID）
```

## 第 3 件：适配云托管 MySQL 环境变量

**当前问题**：`application.yml` 中数据库配置使用的是 `DB_HOST`，但微信云托管使用的是 `MYSQL_ADDRESS`（格式为 `host:port`），需要修改。

> **重要提醒**：MySQL 环境变量（`MYSQL_ADDRESS`、`MYSQL_USERNAME`、`MYSQL_PASSWORD`）**仅在使用官方模板一键部署时自动注入**。手动新建服务并上传代码部署时，必须到「服务设置 → 环境变量」中手动配置这三个变量（连接信息在云托管控制台 → 数据库 → MySQL 页面获取）。

### 操作步骤

修改 `arguing-tutorial-server/src/main/resources/application.yml` 中的数据源配置：

```yaml
spring:
  datasource:
    # 云托管使用 MYSQL_ADDRESS 格式为 host:port，本地默认 localhost:3306
    url: jdbc:mysql://${MYSQL_ADDRESS:localhost:3306}/arguing_tutorial?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
    username: ${MYSQL_USERNAME:root}
    password: ${MYSQL_PASSWORD:root}
```

**改动说明**：
- `DB_HOST` → `MYSQL_ADDRESS`（云托管标准环境变量）
- 用户名从硬编码 `root` → `${MYSQL_USERNAME:root}`
- 密码从 `${DB_PASSWORD:root}` → `${MYSQL_PASSWORD:root}`

> 这是唯一需要改代码的配置项，改完后本地和云端都能正常运行。

## 第 6 件：使用微信云托管部署后端

### 操作步骤

#### 6.1 开通云托管

1. 登录微信小程序后台 https://mp.weixin.qq.com/
2. 云开发 → 云托管 → 开通
3. 创建环境，选择「按量付费」，有 3 个月免费额度

#### 6.2 开通云托管 MySQL

1. 云托管控制台 → 数据库 → 开通 MySQL
2. 记下数据库连接信息（系统会自动注入环境变量，无需手动记录）
3. 用任意 MySQL 客户端连上去（开启外网地址后），执行 `V1__init_schema.sql` 初始化表结构

#### 6.3 本地打包

```bash
cd arguing-tutorial-server
set JAVA_HOME=D:/software/Java/jdk-17
mvn clean package -DskipTests
```

> 确保 `target/` 目录下生成了 `arguing-tutorial-server-0.0.1-SNAPSHOT.jar`。

#### 6.4 创建云托管服务并部署

1. 云托管控制台 → 服务列表 → 新建服务
2. 服务名称：`argue-server`（**最多 20 个字符，必须与前端代码中 `CLOUD_SERVICE_NAME` 一致**）
3. 上传方式：选择「本地上传」
4. 上传整个 `arguing-tutorial-server` 目录（含 Dockerfile、container.config.json、target/）
5. 等待构建和部署完成

#### 6.5 配置环境变量

> **重要**：微信云托管中，MySQL 环境变量**仅在通过官方模板一键部署时自动注入**。如果你是手动新建服务并上传代码部署，**必须手动配置** MySQL 环境变量，否则会因无法连接数据库导致部署失败。
>
> MySQL 连接信息获取方式：云托管控制台 → 数据库 → MySQL 页面，可看到内网地址、用户名、密码。

云托管控制台 → 服务设置 → 环境变量，添加：

```
# ========== 必须手动配置（MySQL 连接信息）==========
# 在云托管控制台 → 数据库 → MySQL 页面获取
MYSQL_ADDRESS=<内网地址>:<端口>    # 例如 10.0.1.100:3306
MYSQL_USERNAME=<用户名>            # 例如 root
MYSQL_PASSWORD=<密码>

# ========== 业务配置 ==========
# AI 服务（智谱 GLM）
AI_API_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
AI_API_KEY=你的智谱API Key
AI_MODEL=glm-4-flash

# 阿里云语音
ALI_ACCESS_KEY_ID=你的阿里云AK
ALI_ACCESS_KEY_SECRET=你的阿里云SK
ALI_APP_KEY=你的语音AppKey

# 腾讯云 COS（对象存储）
COS_SECRET_ID=你的腾讯云SecretId
COS_SECRET_KEY=你的腾讯云SecretKey
COS_REGION=ap-shanghai
COS_BUCKET=7072-prod-3g0nnk2k58e9785d-1421024449
```

> **注意**：`MYSQL_ADDRESS`、`MYSQL_USERNAME`、`MYSQL_PASSWORD` 三个变量必须手动填写，否则后端启动时会报 `Communications link failure` 导致健康检查失败、部署失败。

#### 6.6 验证部署

1. 部署完成后，云托管会分配一个默认域名
2. 测试 `https://默认域名/api/scenes` 应返回场景数据
3. 在云托管控制台查看日志，确认无启动报错

### 8.3 模拟器中验证

- **场景列表**：首页应显示 5 个场景卡片（职场/技术分类）
- **个人中心**：底部 Tab 切换到个人中心，应显示"游客模式"
- **对练入口**：点击场景卡片应能进入对练页

> 如果场景列表为空，打开开发者工具的**调试器（Console 面板）**查看报错日志。

**场景列表不显示的常见原因**：

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 页面显示"暂无场景" | `callContainer` 调用失败 | Console 面板搜索"加载场景列表失败" |
| 页面一直"加载中..." | 网络请求超时 | Network 面板查看请求状态 |
| Console 显示 "wx.cloud is not a function" | 云开发未初始化 | 检查 `manifest.json` 中 `"cloud": true` |
| Console 显示服务不存在 | 云托管服务名不匹配 | 确认控制台服务名为 `argue-server` |

### 8.4 真机测试（语音上传）

模拟器不支持录音，语音功能必须用真机测试。

**方式 A：预览（快速验证）**

1. 开发者工具顶部点 **"预览"** 按钮
2. 弹出二维码，用手机微信扫码
3. 小程序在手机上打开，进入对练页测试录音

**方式 B：真机调试（推荐，能看到日志）**

1. 开发者工具顶部点 **"真机调试"**（在"预览"旁边）
2. 手机扫码打开
3. 手机操作的同时，电脑端开发者工具显示**实时日志**
4. 录音发送后观察 `chat` 接口是否返回 AI 回复

**判断结果**：

| 现象 | 说明 |
|------|------|
| 录音后收到 AI 语音回复 | 语音上传正常，第 8 件通过 |
| 录音后报网络错误 | `callContainer` 的 multipart 上传有问题 |
| 录音后后端返回 500 | 后端音频处理有问题，查看云托管日志 |

如果音频上传失败，可能需要改用以下方案：
- 先将音频上传到 COS，拿到 URL
- 再调用后端接口传递音频 URL（而非文件本身）
