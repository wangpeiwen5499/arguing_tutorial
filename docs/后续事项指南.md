# 程序员吵架修炼场 — 后续事项指南

> 本文档面向后端 Java 开发者，按操作顺序列出每一步需要做什么。

---

## 第 1 件：本地启动 MySQL + Redis

后端依赖 MySQL 和 Redis，开发阶段先在本地装好。

### MySQL

如果本地已有 MySQL，跳过这步。如果没有：

```bash
# 推荐用 Docker
docker run -d --name mysql8 -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_DATABASE=arguing_tutorial mysql:8
```

然后初始化数据库：

```bash
mysql -u root -p123456 arguing_tutorial < arguing-tutorial-server/src/main/resources/db/migration/V1__init_schema.sql
```

### Redis

```bash
docker run -d --name redis -p 6379:6379 redis:7
```

### 操作步骤

1. 确认 MySQL 和 Redis 都在运行
2. 修改 `arguing-tutorial-server/src/main/resources/application.yml` 中的数据库连接信息（用户名、密码）匹配本地配置
3. 在终端运行：

   ```bash
   cd arguing-tutorial-server
   set JAVA_HOME=D:/software/Java/jdk-17
   mvn spring-boot:run
   ```

4. 看到 `Started ArguingTutorialApplication` 表示启动成功
5. 浏览器访问 `http://localhost:8080/api/scenes` 应该能看到 5 条场景数据

## 第 2 件：接入真实大模型 API

当前 `AiService` 已经写好了通用的 LLM 调用逻辑，兼容任何 OpenAI 格式的 API。只需要填入配置。

### 操作步骤

1. **选择一个大模型服务商**，推荐：
   - 国内：DeepSeek（便宜，中文好）、阿里通义千问、智谱 GLM
   - 国外：OpenAI GPT-4o-mini

2. **注册并获取 API Key**（以 DeepSeek 为例）：
   - 访问 https://platform.deepseek.com/
   - 注册账号 → API Keys → 创建一个 Key
   - 复制 Key（形如 `sk-xxxx`）

3. **修改配置**，编辑 `application.yml`：

   ```yaml
   ai:
     api-url: https://api.deepseek.com/v1/chat/completions
     api-key: sk-你的真实key
     model: deepseek-chat
   ```

   或者用环境变量（推荐，不把 key 写进代码）：

   ```bash
   set AI_API_URL=https://api.deepseek.com/v1/chat/completions
   set AI_API_KEY=sk-你的真实key
   set AI_MODEL=deepseek-chat
   mvn spring-boot:run
   ```

4. **验证**：启动后端，用 curl 测试：

   ```bash
   curl -X POST http://localhost:8080/api/auth/guest
   ```

   拿到 token 后测试创建会话，看 AI 是否正常回复。

### 各服务商配置对照

| 服务商 | api-url | model |
|--------|---------|-------|
| DeepSeek | `https://api.deepseek.com/v1/chat/completions` | `deepseek-chat` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | `qwen-turbo` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `glm-4-flash` |
| OpenAI | `https://api.openai.com/v1/chat/completions` | `gpt-4o-mini` |

## 第 3 件：接入真实 ASR + TTS 语音服务

当前 `SpeechService` 是 mock 实现（`recognize` 返回固定文字，`synthesize` 返回 null）。需要接入真实的语音识别和语音合成。

### 操作步骤

1. **选择语音服务商**，推荐阿里云或腾讯云：
   - **阿里云**：语音识别（Paraformer）+ 语音合成（CosyVoice）
   - **腾讯云**：语音识别 + 语音合成

2. **以阿里云为例**：
   - 注册阿里云账号：https://www.aliyun.com/
   - 开通"智能语音交互"服务（有免费额度）
   - 创建项目 → 获取 AccessKey ID 和 AccessKey Secret
   - 获取 AppKey

3. **修改 SpeechService.java**：
   - `pom.xml` 添加阿里云 SDK 依赖
   - `recognize(MultipartFile audioFile)` → 调用阿里云一句话识别 API，传入音频文件，返回识别文字
   - `synthesize(String text)` → 调用阿里云语音合成 API，传入文字，返回音频文件 URL
   - `getPhonemeTimestamps(String text)` → 获取音素时间戳（用于口型驱动）

### 代码改动说明

核心改动在 `SpeechService` 一个文件里，其他代码不用动：

1. 在 `pom.xml` 添加阿里云 SDK 依赖
2. 在 `application.yml` 添加阿里云配置（accessKey、appKey）
3. 重写 `SpeechService` 的三个方法

阿里云官方文档：https://help.aliyun.com/document_detail/84428.html

> 这部分需要写 Java 代码调用 SDK，可以找 Claude 帮忙。

## 第 4 件：注册微信小程序

这一步需要在微信公众平台上操作。

### 操作步骤

1. **注册小程序账号**：
   - 访问 https://mp.weixin.qq.com/
   - 点击"立即注册" → 选择"小程序"
   - 用邮箱注册（不能是已绑定公众号的邮箱）
   - 完成邮箱激活、信息登记（个人即可，选择"个人"主体）
   - 注册完成后登录后台

2. **获取 AppID**：
   - 登录后台 → 开发管理 → 开发设置
   - 找到 AppID（形如 `wx1234567890abcdef`），复制下来

3. **配置服务器域名**（后端部署后再做，先记下）：
   - 开发设置 → 服务器域名
   - request 合法域名：添加后端地址（如 `https://api.yourdomain.com`）
   - uploadFile 合法域名：同上
   - downloadFile 合法域名：同上

4. **下载微信开发者工具**：
   - 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 下载 Windows 稳定版，安装

> 纯网页操作，不需要写代码。注册完后把 AppID 记下来。

## 第 6 件：部署后端到服务器

小程序要求后端必须是 `https://` 开头的域名，不能是 localhost。需要一台服务器。

### 操作步骤

1. **买一台云服务器**（阿里云/腾讯云轻量应用服务器即可，最低配够用）
   - 推荐配置：2核2G，约 50-100 元/月

2. **买一个域名**（已有则跳过）
   - 阿里云/腾讯云都可以买，`.com` 域名约 60 元/年
   - 在云服务商后台做域名备案（个人备案，约 7-15 天）

3. **部署后端**：

   ```bash
   # 在本地打包
   cd arguing-tutorial-server
   set JAVA_HOME=D:/software/Java/jdk-17
   mvn clean package -DskipTests

   # 上传 target/*.jar 到服务器（用 scp 或 FTP 工具）

   # 在服务器上安装 JDK 17 + MySQL + Redis（参考第 1 件）
   # 初始化数据库（参考第 1 件）

   # 在服务器上运行
   java -jar arguing-tutorial-server-1.0.0.jar \
     --spring.datasource.url=jdbc:mysql://localhost:3306/arguing_tutorial \
     --spring.datasource.password=你的密码 \
     --ai.api-key=你的AI密钥
   ```

4. **配置 HTTPS**：
   - 用 Nginx 做反向代理
   - 用 Let's Encrypt 免费证书（或云服务商免费证书）
   - 域名如 `api.yourdomain.com` 指向服务器 IP

   Nginx 配置示例：

   ```nginx
   server {
       listen 443 ssl;
       server_name api.yourdomain.com;

       ssl_certificate /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem;
       ssl_certificate_key /etc/letsencrypt/live/api.yourdomain.com/privkey.pem;

       location / {
           proxy_pass http://127.0.0.1:8080;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           client_max_body_size 10m;  # 允许上传音频文件
       }
   }
   ```

> 需要买服务器 + 买域名 + 备案 + 配置 Nginx + 部署 jar 包。都是运维操作，不需要写代码但需要 Linux 命令。如果不熟可以找 Claude 一步步指导。

   const BASE_URL = 'https://api.yourdomain.com'

## 第 8 件：接入微信登录

当前 `AuthService.loginByWx` 是 mock 实现。需要替换为真实微信 API 调用。

### 操作步骤

1. 在微信公众平台获取 AppSecret：
   - 开发管理 → 开发设置 → AppSecret（重置后复制，只显示一次）

2. 修改 `application.yml` 添加微信配置：

   ```yaml
   wechat:
     appid: wx你的appid
     secret: 你的appsecret
   ```

3. 修改 `AuthService.java` 的 `loginByWx` 方法：

   ```java
   public User loginByWx(String code) {
       // 1. 调用微信 API 换取 openid
       String url = String.format(
           "https://api.weixin.qq.com/sns/jscode2session?appid=%s&secret=%s&js_code=%s&grant_type=authorization_code",
           appid, secret, code
       );
       // 2. 用 RestTemplate 调用，解析 JSON 获取 openid
       // 3. 查找或创建用户
       // 4. 返回用户
   }
   ```

   大概 20 行代码。

4. 改完后重新 `mvn clean package -DskipTests` 打包部署

> 需要写一点 Java 代码。改动集中在 `AuthService.java`，可以找 Claude 帮忙写。

### 操作步骤

1. **开通对象存储服务**：
   - 阿里云 OSS 或腾讯云 COS（有免费额度）
   - 创建一个 Bucket（如 `arguing-tutorial-assets`）
   - 设置为公共读、私有写

2. **修改代码**：
   - `pom.xml` 添加 OSS SDK 依赖
   - 修改 `SpeechService.synthesize` 的返回值为 OSS URL
   - 修改 `ShareService.generateShareCard` 上传到 OSS
   - 修改 `SessionService.chat` 保存用户音频到 OSS

3. **配置跨域**：
   - OSS Bucket 设置跨域规则，允许 `https://api.yourdomain.com` 访问

> 需要写 Java 代码调用 OSS SDK（上传文件）。改动集中在 `SpeechService` 和 `ShareService`，可以找 Claude 帮忙写。

## 优先级总览

| 优先级 | 事项 | 需要做什么 | 是否需要写代码 |
|--------|------|-----------|--------------|
| **P0** | MySQL + Redis 本地启动 | 装 Docker / 改配置 | 否 |
| **P0** | 接入大模型 API | 注册拿 Key / 改配置 | 否（配置已写好） |
| **P1** | 接入 ASR + TTS | 注册云服务 / 写代码 | **是**（SpeechService） |
| **P1** | 注册微信小程序 | 网页操作 | 否 |
| **P2** | 部署后端 | 买服务器 / 配 Nginx | 否（运维操作） |
| **P2** | 配置小程序域名 | 网页操作 + 改一行配置 | 否 |
| **P2** | 微信登录对接 | 写代码 | **是**（AuthService） |
| **P3** | 接入 CDN | 注册云服务 / 写代码 | **是**（SpeechService + ShareService） |

### 建议执行顺序

1. 先从 **P0** 开始：本地跑通 MySQL + Redis + 大模型 API，确认后端能正常返回 AI 回复
2. 再做 **P1**：接入语音服务 + 注册小程序，本地能看到完整效果
3. 然后 **P2**：部署上线 + 配域名 + 微信登录
4. 最后 **P3**：CDN 接入，优化体验

需要写代码的部分（ASR/TTS、微信登录、CDN），可以随时找 Claude 帮忙。
