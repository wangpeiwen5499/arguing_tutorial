> 本文档面向后端 Java 开发者，按操作顺序列出每一步需要做什么。

## 第 1 件：本地启动 MySQL + Redis（开发调试用）

后端依赖 MySQL 和 Redis。部署到微信云托管后，云托管会提供 MySQL，但本地开发调试仍需自己装。

> 如果只打算用云托管、不需要本地调试，可以跳过这步。

### MySQL

如果本地已有 MySQL，跳过这步。如果没有：

```bash
# 推荐用 Docker
docker run -d --name mysql8 -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_DATABASE=arguing_tutorial mysql:8
```

然后初始化数据库：

```bash
mysql -u root -p123456 arguing_tutorial < arguing-tutorial-server/src/main/resources/db/migration/V1__init_schema.sql
```

### Redis

```bash
docker run -d --name redis -p 6379:6379 redis:7
```

### 操作步骤

1. 确认 MySQL 和 Redis 都在运行
2. 修改 `arguing-tutorial-server/src/main/resources/application.yml` 中的数据库连接信息（用户名、密码）匹配本地配置
3. 在终端运行：

   ```bash
   cd arguing-tutorial-server
   set JAVA_HOME=D:/software/Java/jdk-17
   mvn spring-boot:run
   ```

4. 看到 `Started ArguingTutorialApplication` 表示启动成功
5. 浏览器访问 `http://localhost:8080/api/scenes` 应该能看到 5 条场景数据

## 第 2 件：接入真实大模型 API

当前 `AiService` 已经写好了通用的 LLM 调用逻辑，兼容任何 OpenAI 格式的 API。只需要填入配置。

### 操作步骤

1. **选择一个大模型服务商**，推荐：
   - 国内：DeepSeek（便宜，中文好）、阿里通义千问、智谱 GLM
   - 国外：OpenAI GPT-4o-mini

2. **注册并获取 API Key**（以 DeepSeek 为例）：
   - 访问 https://platform.deepseek.com/
   - 注册账号 → API Keys → 创建一个 Key
   - 复制 Key（形如 `sk-xxxx`）

3. **修改配置**，编辑 `application.yml`：

   ```yaml
   ai:
     api-url: https://api.deepseek.com/v1/chat/completions
     api-key: sk-你的真实key
     model: deepseek-chat
   ```

   或者用环境变量（推荐，不把 key 写进代码）：

   ```bash
   set AI_API_URL=https://api.deepseek.com/v1/chat/completions
   set AI_API_KEY=sk-你的真实key
   set AI_MODEL=deepseek-chat
   mvn spring-boot:run
   ```

4. **验证**：启动后端，用 curl 测试：

   ```bash
   curl -X POST http://localhost:8080/api/auth/guest
   ```

   拿到 token 后测试创建会话，看 AI 是否正常回复。

### 各服务商配置对照

| 服务商 | api-url | model |
|--------|---------|-------|
| DeepSeek | `https://api.deepseek.com/v1/chat/completions` | `deepseek-chat` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | `qwen-turbo` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `glm-4-flash` |
| OpenAI | `https://api.openai.com/v1/chat/completions` | `gpt-4o-mini` |

## 第 3 件：接入真实 ASR + TTS 语音服务

当前 `SpeechService` 是 mock 实现（`recognize` 返回固定文字，`synthesize` 返回 null）。需要接入真实的语音识别和语音合成。

### 操作步骤

1. **选择语音服务商**，推荐阿里云或腾讯云：
   - **阿里云**：语音识别（Paraformer）+ 语音合成（CosyVoice）
   - **腾讯云**：语音识别 + 语音合成

2. **以阿里云为例**：
   - 注册阿里云账号：https://www.aliyun.com/
   - 开通"智能语音交互"服务（有免费额度）
   - 创建项目 → 获取 AccessKey ID 和 AccessKey Secret
   - 获取 AppKey

3. **修改 SpeechService.java**：
   - `pom.xml` 添加阿里云 SDK 依赖
   - `recognize(MultipartFile audioFile)` → 调用阿里云一句话识别 API，传入音频文件，返回识别文字
   - `synthesize(String text)` → 调用阿里云语音合成 API，传入文字，返回音频文件 URL
   - `getPhonemeTimestamps(String text)` → 获取音素时间戳（用于口型驱动）

### 代码改动说明

核心改动在 `SpeechService` 一个文件里，其他代码不用动：

1. 在 `pom.xml` 添加阿里云 SDK 依赖
2. 在 `application.yml` 添加阿里云配置（accessKey、appKey）
3. 重写 `SpeechService` 的三个方法

阿里云官方文档：https://help.aliyun.com/document_detail/84428.html

> 这部分需要写 Java 代码调用 SDK，可以找 Claude 帮忙。

## 第 4 件：注册微信小程序

这一步需要在微信公众平台上操作。

### 操作步骤

1. **注册小程序账号**：
   - 访问 https://mp.weixin.qq.com/
   - 点击"立即注册" → 选择"小程序"
   - 用邮箱注册（不能是已绑定公众号的邮箱）
   - 完成邮箱激活、信息登记（个人即可，选择"个人"主体）
   - 注册完成后登录后台

2. **获取 AppID**：
   - 登录后台 → 开发管理 → 开发设置
   - 找到 AppID（形如 `wx1234567890abcdef`），复制下来

3. **配置服务器域名**（后端部署后再做，先记下）：
   - 开发设置 → 服务器域名
   - request 合法域名：添加后端地址（如 `https://api.yourdomain.com`）
   - uploadFile 合法域名：同上
   - downloadFile 合法域名：同上

4. **下载微信开发者工具**：
   - 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
   - 下载 Windows 稳定版，安装

> 纯网页操作，不需要写代码。注册完后把 AppID 记下来。

## 第 6 件：使用微信云托管部署后端

使用微信云托管可以免买服务器、免买域名、免备案、免配 HTTPS，直接把后端部署到微信的容器云上。

> 注意：微信云托管（容器服务）和微信云开发（TCB）是两个产品。Spring Boot 项目应选择**微信云托管**。

### 操作步骤

1. **开通微信云托管**：
   - 登录微信小程序后台 https://mp.weixin.qq.com/
   - 云开发 → 云托管 → 开通（如果还没有服务号，按提示关联）
   - 选择「按量付费」，有免费额度

2. **开通云托管 MySQL**：
   - 云托管控制台 → 数据库 → 开通 MySQL
   - 记下数据库连接地址、用户名、密码
   - 用任意 MySQL 客户端连上去，执行 `V1__init_schema.sql` 初始化表结构

3. **准备部署文件**：

   在 `arguing-tutorial-server/` 目录下新建 `Dockerfile`：

   ```dockerfile
   FROM eclipse-temurin:17-jre
   WORKDIR /app
   COPY target/*.jar app.jar
   EXPOSE 80
   ENTRYPOINT ["java", "-jar", "app.jar"]
   ```

   新建 `container.config.json`：

   ```json
   {
     "containerPort": 80,
     "minNum": 0,
     "maxNum": 5,
     "cpu": 0.5,
     "mem": 512
   }
   ```

   修改 `application.yml` 中的 server.port 为 80：

   ```yaml
   server:
     port: 80
   ```

4. **本地打包**：

   ```bash
   cd arguing-tutorial-server
   set JAVA_HOME=D:/software/Java/jdk-17
   mvn clean package -DskipTests
   ```

5. **创建云托管服务并部署**：
   - 云托管控制台 → 服务列表 → 新建服务
   - 服务名称：`arguing-tutorial-server`
   - 上传方式：选择「本地上传」或「代码仓库」
   - 如果用本地上传：上传整个 `arguing-tutorial-server` 目录（含 Dockerfile）
   - 如果用代码仓库：绑定 GitHub/GitLab 仓库，设置自动部署

6. **配置环境变量**：

   在云托管服务设置 → 环境变量中添加：

   ```
   DB_PASSWORD=云托管MySQL的密码
   DB_HOST=云托管MySQL的内网地址
   AI_API_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
   AI_API_KEY=你的AI密钥
   AI_MODEL=glm-4-flash
   ALI_ACCESS_KEY_ID=你的阿里云AK
   ALI_ACCESS_KEY_SECRET=你的阿里云SK
   ALI_APP_KEY=你的语音AppKey
   ```

   同时修改 `application.yml` 中数据库 URL 引用环境变量：

   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://${DB_HOST:localhost}:3306/arguing_tutorial?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false
   ```

7. **验证部署**：
   - 部署完成后，云托管会分配一个默认域名（形如 `xxx.ap-shanghai.run.tcloudbase.com`）
   - 点击服务详情里的「默认域名」可以访问
   - 测试 `https://默认域名/api/scenes` 应返回场景数据

> 不需要买服务器、买域名、备案、配 Nginx。全部在微信云托管网页操作完成。

## 第 7 件：前端改用云托管调用方式

后端部署到云托管后，小程序前端需要改用 `wx.cloud.callContainer` 调用接口，而不是传统的 `wx.request`。

> 云托管走微信内网通道，不需要配置「服务器域名」，也没有域名白名单限制。

### 操作步骤

1. **前端初始化云托管 SDK**：

   修改 `arguing-tutorial-client/src/api/index.js`（或对应的 API 请求封装文件）：

   ```javascript
   // 用云托管调用替代 wx.request
   const callCloud = (options) => {
     return wx.cloud.callContainer({
       name: 'arguing-tutorial-server', // 云托管服务名
       path: options.url,
       method: options.method || 'GET',
       data: options.data,
       header: {
         'X-WX-OPENID': '{openid}', // 自动获取用户 OpenID
         ...options.header
       }
     })
   }

   // 示例：获取场景列表
   export const getScenes = () => {
     return callCloud({
       url: '/api/scenes',
       method: 'GET'
     })
   }
   ```

2. **确保小程序已开通云开发**：
   - 小程序后台 → 云开发 → 开通（免费版即可，只用来做云托管的入口）
   - 在 `app.json` 中添加：
     ```json
     "cloud": true
     ```
   - 在 `App.onLaunch` 中初始化：
     ```javascript
     if (wx.cloud) {
       wx.cloud.init()
     }
     ```

3. **重新编译前端**：

   ```bash
   cd arguing-tutorial-client
   npm run build:mp-weixin
   ```

4. 在微信开发者工具中测试，确认接口能正常调通

> 改动集中在前端 API 调用层，后端代码基本不需要改。使用云托管后无需配置服务器域名白名单。

## 第 8 件：接入微信登录（云托管简化版）

使用云托管后，微信登录大幅简化——云托管会自动在请求头中注入用户的 OpenID，不需要再调用微信的 `jscode2session` 接口。

### 操作步骤

1. **后端读取 OpenID**：

   在 `AuthService.java` 中，从请求头获取 OpenID：

   ```java
   public User loginByWx(String code) {
       // 云托管模式下，OpenID 由云托管自动注入到请求头
       // 不需要调用微信 jscode2session 接口
       String openid = request.getHeader("X-WX-OPENID");
       if (openid == null || openid.isEmpty()) {
           throw new RuntimeException("无法获取用户 OpenID");
       }
       // 查找或创建用户
       User user = userRepository.findByOpenid(openid);
       if (user == null) {
           user = new User();
           user.setOpenid(openid);
           user.setNickname("用户" + openid.substring(0, 6));
           user = userRepository.save(user);
       }
       return user;
   }
   ```

2. **前端登录调用**：

   ```javascript
   // 前端只需调用云托管接口，OpenID 会自动带上
   const login = () => {
     return callCloud({
       url: '/api/auth/guest',
       method: 'POST'
     })
   }
   ```

3. 改完后重新打包部署：`mvn clean package -DskipTests`，然后在云托管控制台重新上传

> 不需要 AppSecret，不需要调微信 API，不需要额外的配置。云托管自动完成用户身份识别。改动很小，大概 10 行代码。

## 优先级总览

| 优先级 | 事项 | 需要做什么 | 是否需要写代码 |
|--------|------|-----------|--------------|
| **P0** | MySQL + Redis 本地启动 | 装 Docker / 改配置（本地调试用） | 否 |
| **P0** | 接入大模型 API | 注册拿 Key / 改配置 | 否（配置已写好） |
| **P1** | 接入 ASR + TTS | 注册云服务 / 写代码 | **是**（SpeechService） |
| **P1** | 注册微信小程序 | 网页操作 | 否 |
| **P1** | 使用微信云托管部署 | 网页操作 + 准备 Dockerfile | 少量（Dockerfile + 环境变量） |
| **P1** | 前端改用云托管调用 | 改 API 调用方式 | **是**（前端 API 层） |
| **P2** | 微信登录（云托管简化版） | 后端读请求头 | **是**（AuthService，约10行） |
| **P3** | 接入对象存储 | 开通存储 / 改代码 | **是**（SpeechService + ShareService） |

### 建议执行顺序

1. 先从 **P0** 开始：本地跑通 MySQL + Redis + 大模型 API，确认后端能正常返回 AI 回复
2. 再做 **P1**：接入语音服务 + 注册小程序 + 部署到云托管 + 前端改云托管调用
3. 然后 **P2**：对接微信登录（云托管简化版，改动很小）
4. 最后 **P3**：对象存储接入，优化体验

需要写代码的部分（ASR/TTS、云托管调用、微信登录、对象存储），可以随时找 Claude 帮忙。
