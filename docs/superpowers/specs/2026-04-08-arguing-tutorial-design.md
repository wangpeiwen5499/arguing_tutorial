# 程序员吵架修炼场 — 产品设计文档

## 概述

面向中文程序员的谈判与辩论训练小程序。核心体验是用户与 AI 数字人进行全语音沉浸式对练，结束后获得多维度复盘分析。支持预设场景和自定义场景的混合模式。

## 目标用户

中文程序员群体，希望提升职场谈判、技术辩论、日常沟通能力。

## 产品形态

微信小程序，快速体验优先（无需注册即可开始对练），注册后可保存历史记录和进度。

## 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | uni-app（Vue 语法，微信小程序） |
| 后端 | Spring Boot |
| 数字人 | 预制 2D/3D 模型 + 口型/表情驱动 |
| AI | 大模型 API（角色扮演 + 对话 + 复盘分析） |
| 语音 | ASR（语音识别）+ TTS（语音合成） |

## 产品方案：混合模式

提供精选场景库作为引导，同时支持"自定义场景"模式。场景库按职场类、技术类、日常类分类。初期实现预设场景，后续迭代加入自定义场景。

## 核心用户流程

### 1. 首页 → 快速开始

- 打开小程序即看到热门场景卡片（"谈加薪"、"怼不合理需求"、"代码评审辩论"等）
- 点击任意卡片立即进入对练，无需注册
- 提供"自定义场景"入口，用户自由描述情境

### 2. 沉浸式对练

全屏沉浸式界面，核心要素：

- **场景化背景**：全屏背景匹配场景氛围（老板办公室、开放工位、会议室等）
- **数字人上半身**：预制 2D/3D 角色模型，实时口型同步 + 表情变化
- **全语音交互**：
  - 用户按住麦克风说话，松开结束（按键说话模式）
  - AI 语音回复（TTS），文字仅做底部小字幕弱化展示
- **情绪联动**：数字人表情和语气随对话进展实时变化（生气/嘲讽/犹豫/妥协）
- 对话轮次有上限（10 轮），侧边有策略提示（使用会扣分）

### 3. 复盘分析

对练结束后自动生成复盘报告：

- **多维度评分**：逻辑性、情绪控制、说服力、策略运用、表达清晰度
- **总体评分** + 与上次对比
- **做得好的** / **可以改进的** 具体点评
- **每轮对话的逐一复盘**
- 支持"再来一次"或"换策略重试"

### 4. 社交分享

练完后生成战绩分享卡片：

- 展示场景名称、综合评分、关键维度星级
- 带小程序码，好友扫码直接挑战同场景
- 支持分享到微信好友 / 朋友圈 / 群聊

## 技术架构

### 系统架构

前端（uni-app 小程序）通过 REST API 与 Spring Boot 后端通信。后端统一管理 AI 调用、语音处理、数字人驱动数据生成。

### 核心服务

| 服务 | 职责 |
|------|------|
| 场景服务 | 场景 CRUD、难度配置、角色设定 |
| 对练服务 | 对话管理、AI Prompt 编排、轮次控制 |
| 语音服务 | ASR 语音识别、TTS 语音合成 |
| 数字人服务 | 模型加载、口型/表情驱动数据生成 |
| 分析服务 | 复盘报告生成、多维度打分 |
| 用户服务 | 微信授权登录、历史记录、进度追踪 |
| 分享服务 | 战绩卡片生成、分享配置 |

### 核心数据模型

```
场景（Scene）
├── id, name, description
├── category（职场/技术/日常）
├── difficulty（1-5星）
├── background_config（背景图/氛围配置）
├── avatar_config（数字人模型ID + 外观参数）
├── personality（对手性格设定：强硬/温和/狡猾等）
├── opening_line（开场白）
└── evaluation_criteria（评分标准权重）

对练会话（Session）
├── id
├── user_id（可为空，未登录用户）
├── scene_id
├── status（进行中/已完成/放弃）
├── total_rounds, current_round
├── hint_used_count（使用提示次数）
└── created_at, finished_at

对话轮次（Round）
├── id, session_id
├── round_number
├── user_audio_url（用户语音文件）
├── user_text（ASR 识别结果）
├── ai_text（AI 回复文本）
├── ai_audio_url（TTS 语音文件）
├── ai_emotion（AI 当前情绪标记）
├── ai_expression（表情/动作指令）
└── timestamp

复盘报告（Report）
├── id, session_id
├── total_score（总分）
├── logic_score, emotion_score, persuasion_score...
├── strengths（做得好的点，数组）
├── improvements（可改进的点，数组）
├── round_reviews（每轮点评，数组）
└── share_card_url（分享卡片图）
```

### 核心 API

```
场景相关：
GET    /api/scenes              # 场景列表（分页、分类筛选）
GET    /api/scenes/{id}         # 场景详情
POST   /api/scenes/custom       # 创建自定义场景

对练相关：
POST   /api/sessions            # 开始对练（传 scene_id）
POST   /api/sessions/{id}/chat  # 发送用户语音，返回 AI 回复 + 语音 + 表情指令
POST   /api/sessions/{id}/hint  # 请求策略提示
POST   /api/sessions/{id}/end   # 结束对练，触发复盘分析

复盘相关：
GET    /api/reports/{sessionId} # 获取复盘报告
GET    /api/reports/{sessionId}/share-card  # 生成分享卡片

用户相关：
POST   /api/auth/wx-login      # 微信授权登录
GET    /api/user/profile        # 个人信息
GET    /api/user/history        # 对练历史
GET    /api/user/stats          # 统计数据（总分/场次/趋势）
```

### 对练核心时序

```
用户按住说话 → 前端录音
    → 松开，上传音频到后端
        → 后端 ASR 转文字
        → 文字 + 上下文 + 角色 Prompt → 调用大模型
            → 返回 AI 回复文本 + 情绪标记 + 表情指令
        → AI 文本 → TTS 生成语音
        → 返回前端：{ text, audio_url, emotion, expression }
    → 前端播放语音 + 驱动数字人口型/表情
    → 进入下一轮
```

## 关键页面

1. **首页** — 热门场景卡片列表 + 自定义场景入口
2. **沉浸式对练界面** — 全屏场景背景 + 数字人 + 语音交互 + 弱化文字字幕
3. **复盘报告** — 多维度评分 + 做得好/可改进点评 + 每轮复盘
4. **分享卡片** — 成绩单式社交分享，带小程序码
5. **个人中心** — 对练历史 / 进度追踪 / 统计数据

## 设计原则

- **沉浸感优先**：全语音 + 数字人 + 场景化背景，打造真实对练体验
- **快速体验**：无需注册即可开始，降低使用门槛
- **社交裂变**：分享战绩激发好友挑战，驱动增长
- **渐进式复杂度**：MVP 先做预设场景 + 预制数字人，验证后迭代
