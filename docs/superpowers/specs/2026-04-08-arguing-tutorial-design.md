# 程序员吵架修炼场 — 产品设计文档

## 概述

面向中文程序员的谈判与辩论训练小程序。核心体验是用户与 AI 数字人进行全语音沉浸式对练，结束后获得多维度复盘分析。支持预设场景和自定义场景的混合模式。

## 目标用户

中文程序员群体，希望提升职场谈判、技术辩论、日常沟通能力。

## 产品形态

微信小程序，快速体验优先（无需注册即可开始对练），注册后可保存历史记录和进度。

## 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | uni-app（Vue 语法，微信小程序） |
| 后端 | Spring Boot |
| 数字人 | 预制 2D/3D 模型 + 口型/表情驱动 |
| AI | 大模型 API（角色扮演 + 对话 + 复盘分析） |
| 语音 | ASR（语音识别）+ TTS（语音合成） |

## 产品方案：混合模式

提供精选场景库作为引导，同时支持"自定义场景"模式。场景库按职场类、技术类、日常类分类。初期实现预设场景，后续迭代加入自定义场景。

## 核心用户流程

### 1. 首页 → 快速开始

- 打开小程序即看到热门场景卡片（"谈加薪"、"怼不合理需求"、"代码评审辩论"等）
- 点击任意卡片立即进入对练，无需注册
- 提供"自定义场景"入口，用户自由描述情境

### 2. 沉浸式对练

全屏沉浸式界面，核心要素：

- **场景化背景**：全屏背景匹配场景氛围（老板办公室、开放工位、会议室等）
- **数字人上半身**：预制 2D/3D 角色模型，实时口型同步 + 表情变化
- **全语音交互**：
  - 用户按住麦克风说话，松开结束（按键说话模式）
  - AI 语音回复（TTS），文字仅做底部小字幕弱化展示
- **情绪联动**：数字人表情和语气随对话进展实时变化（生气/嘲讽/犹豫/妥协）
- 对话轮次有上限（10 轮），侧边有策略提示（使用会扣分）

### 3. 复盘分析

对练结束后自动生成复盘报告：

- **多维度评分**：逻辑性、情绪控制、说服力、策略运用、表达清晰度
- **总体评分** + 与上次对比
- **做得好的** / **可以改进的** 具体点评
- **每轮对话的逐一复盘**
- 支持"再来一次"或"换策略重试"

### 4. 社交分享

练完后生成战绩分享卡片：

- 展示场景名称、综合评分、关键维度星级
- 带小程序码，好友扫码直接挑战同场景
- 支持分享到微信好友 / 朋友圈 / 群聊

## 技术架构

### 系统架构

前端（uni-app 小程序）通过 REST API 与 Spring Boot 后端通信。后端统一管理 AI 调用、语音处理、数字人驱动数据生成。

### 核心服务

| 服务 | 职责 |
|------|------|
| 场景服务 | 场景 CRUD、难度配置、角色设定 |
| 对练服务 | 对话管理、AI Prompt 编排、轮次控制 |
| 语音服务 | ASR 语音识别、TTS 语音合成 |
| 数字人服务 | 模型加载、口型/表情驱动数据生成 |
| 分析服务 | 复盘报告生成、多维度打分 |
| 用户服务 | 微信授权登录、历史记录、进度追踪 |
| 分享服务 | 战绩卡片生成、分享配置 |

### 核心数据模型

```
场景（Scene）
├── id, name, description
├── category（职场/技术/日常）
├── difficulty（1-5星）
├── background_config（背景图/氛围配置）
├── avatar_config（数字人模型ID + 外观参数）
├── personality（对手性格设定：强硬/温和/狡猾等）
├── opening_line（开场白）
└── evaluation_criteria（评分标准权重）

对练会话（Session）
├── id
├── user_id（可为空，未登录用户）
├── scene_id
├── status（进行中/已完成/放弃）
├── total_rounds, current_round
├── hint_used_count（使用提示次数）
└── created_at, finished_at

对话轮次（Round）
├── id, session_id
├── round_number
├── user_audio_url（用户语音文件）
├── user_text（ASR 识别结果）
├── ai_text（AI 回复文本）
├── ai_audio_url（TTS 语音文件）
├── ai_emotion（AI 当前情绪标记）
├── ai_expression（表情/动作指令）
└── timestamp

复盘报告（Report）
├── id, session_id
├── total_score（总分）
├── logic_score, emotion_score, persuasion_score...
├── strengths（做得好的点，数组）
├── improvements（可改进的点，数组）
├── round_reviews（每轮点评，数组）
└── share_card_url（分享卡片图）
```

### 核心 API

```
场景相关：
GET    /api/scenes              # 场景列表（分页、分类筛选）
GET    /api/scenes/{id}         # 场景详情
POST   /api/scenes/custom       # 创建自定义场景

对练相关：
POST   /api/sessions            # 开始对练（传 scene_id）
POST   /api/sessions/{id}/chat  # 发送用户语音，返回 AI 回复 + 语音 + 表情指令
POST   /api/sessions/{id}/hint  # 请求策略提示
POST   /api/sessions/{id}/end   # 结束对练，触发复盘分析

复盘相关：
GET    /api/reports/{sessionId} # 获取复盘报告
GET    /api/reports/{sessionId}/share-card  # 生成分享卡片

用户相关：
POST   /api/auth/wx-login      # 微信授权登录
GET    /api/user/profile        # 个人信息
GET    /api/user/history        # 对练历史
GET    /api/user/stats          # 统计数据（总分/场次/趋势）
```

### 对练核心时序

```
用户按住说话 → 前端录音
    → 松开，上传音频到后端
        → 后端 ASR 转文字
        → 文字 + 上下文 + 角色 Prompt → 调用大模型
            → 返回 AI 回复文本 + 情绪标记 + 表情指令
        → AI 文本 → TTS 生成语音
        → 返回前端：{ text, audio_url, emotion, expression }
    → 前端播放语音 + 驱动数字人口型/表情
    → 进入下一轮
```

## 关键页面

1. **首页** — 热门场景卡片列表 + 自定义场景入口
2. **沉浸式对练界面** — 全屏场景背景 + 数字人 + 语音交互 + 弱化文字字幕
3. **复盘报告** — 多维度评分 + 做得好/可改进点评 + 每轮复盘
4. **分享卡片** — 成绩单式社交分享，带小程序码
5. **个人中心** — 对练历史 / 进度追踪 / 统计数据

## 音频约束

| 参数 | 值 |
|------|---|
| 最短录音时长 | 1 秒（低于则忽略，提示用户重试） |
| 最长录音时长 | 60 秒（到时自动截止并上传） |
| 音频格式 | AAC（微信 RecorderManager 默认格式） |
| 采样率 | 16000 Hz |
| 最大文件大小 | 2 MB |
| 取消手势 | 上滑取消（录音时上滑显示"松开取消"） |

## 游客会话管理

- **身份标识**：首次打开小程序时由后端生成临时 token，存储在小程序 Storage 中
- **会话持久化**：游客会话保存 7 天，过期自动清理
- **功能限制**：游客可使用全部对练功能，但历史记录最多保留 3 条
- **游客转注册**：注册时自动将游客会话迁移到正式账号，历史记录保留
- **频率限制**：游客每天最多 5 次对练，注册用户无限制

## 轮次与结束行为

- 每场对练固定 10 轮
- 用户可在任意轮次主动结束（界面有"结束对练"按钮）
- 第 10 轮结束后自动触发复盘分析
- AI Prompt 中包含剩余轮次信息，AI 在后几轮会主动推动对话走向结论
- 界面顶部显示进度条（当前轮次/总轮次）

## 策略提示

- 每场对练最多 3 次提示机会
- 提示内容为当前情境下的建议策略名称 + 一句话说明（如"数据举证：用具体数字支撑你的论点"）
- 不提供完整回复建议，只给方向性提示
- 每使用一次提示，总分扣 5 分（满分 100）

## 复盘评分体系

- **评分范围**：0-100 分
- **五个维度**，各占不同权重：

| 维度 | 权重 | 说明 |
|------|------|------|
| 逻辑性 | 25% | 论点是否有逻辑支撑，是否存在逻辑漏洞 |
| 情绪控制 | 20% | 是否保持冷静，有无情绪化表达 |
| 说服力 | 25% | 论据是否充分，能否打动对方 |
| 策略运用 | 15% | 是否灵活运用谈判策略（共情、施压、妥协等） |
| 表达清晰度 | 15% | 表达是否简洁清晰，有无冗余 |

- **总分计算**：各维度得分 × 权重之和，再减去提示扣分
- **维度评分**由大模型基于评分标准 Prompt 生成，每轮独立评分，最终取加权平均
- **Scene.evaluation_criteria** 可覆盖默认权重（格式：`{"logic": 0.3, "emotion": 0.15, ...}`）

## 数字人技术方案

- **渲染引擎**：Canvas 2D（兼容性最佳，性能可控）
- **模型格式**：Live2D 风格的 2D 模型，JSON 配置 + 纹理图集
- **口型驱动**：基于 TTS 返回的音素时间戳映射到口型关键帧（a/i/u/e/o 5 种口型）
- **表情驱动**：AI 返回 emotion 标记（angry/sarcastic/hesitant/compromising/confident），前端映射到对应表情动画
- **资源加载**：模型资产存 CDN，按场景懒加载（进入对练前预加载对应角色）
- **包体积**：小程序主体控制在 2MB 以内，数字人资源全部走 CDN 动态加载

## 通信与延迟

- **协议**：对练核心流程使用 REST（请求-响应模式），语音生成完成后一次性返回
- **延迟目标**：每轮总响应时间 ≤ 5 秒（ASR 1s + LLM 2s + TTS 1s + 网络 1s）
- **等待状态**：处理期间数字人展示"思考中"动画（轻微呼吸 + 眨眼 + 皱眉），底部显示波形加载指示
- **超时处理**：单次请求超时 10 秒，超时后自动重试一次，仍失败则提示用户重试

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| ASR 识别失败/为空 | 提示"没听清，再说一次"，不消耗轮次 |
| 录音时间过短（<1s） | 提示"说话时间太短"，不消耗轮次 |
| LLM 调用超时 | 自动重试 1 次，仍失败则提示"对手走神了，再来一次" |
| TTS 生成失败 | 降级为纯文字回复（无语音，仅字幕） |
| 网络断开 | 显示断网提示，会话状态保留 30 分钟，恢复后可继续 |
| 微信授权被拒 | 游客模式正常使用，个人中心功能受限 |

## 内容安全

- **用户输入过滤**：ASR 文本经敏感词过滤后再传入 LLM
- **AI 输出审核**：LLM 回复经内容安全审核后再返回前端
- **角色 Prompt 约束**：系统 Prompt 中明确要求角色扮演冲突场景但不生成侮辱、歧视、暴力等有害内容
- **合规要求**：遵守《生成式人工智能服务管理暂行办法》和《互联网信息服务深度合成管理规定》

## 自定义场景

用户创建自定义场景只需提供：
- `name`：场景名称
- `description`：情境描述（自由文本，如"我是后端开发，前端同事非要我在接口里加业务逻辑"）
- `opponent_description`：对手描述（如"一个强势的前端老员工"）

系统自动处理：
- `avatar_config`：根据 opponent_description 选择最匹配的预制模型
- `personality`：由 LLM 根据 description 生成对手性格设定
- `opening_line`：由 LLM 生成开场白
- `evaluation_criteria`：使用默认评分权重
- `difficulty`：由 LLM 根据情境复杂度自动评估

## 分享卡片

- **生成方式**：后端使用模板渲染为图片（服务端 Canvas）
- **图片规格**：750 × 600 像素，JPG 格式
- **存储**：上传至 CDN，7 天过期
- **小程序码**：调用微信 API 生成，嵌入卡片右下角
- **分享入口**：复盘报告页"分享战绩"按钮，调用 `wx.shareAppMessage` / `wx.shareTimeline`

## 微信小程序约束

- 录音使用 `wx.getRecorderManager()`，需用户授权 `scope.record`
- 音频播放使用 `InnerAudioContext`
- Canvas 2D 渲染数字人，需测试低端机性能
- 主体包 ≤ 2MB，分包 ≤ 20MB，数字人资源全部走 CDN
- 遵循小程序审核规范，避免敏感内容

## 设计原则

- **沉浸感优先**：全语音 + 数字人 + 场景化背景，打造真实对练体验
- **快速体验**：无需注册即可开始，降低使用门槛
- **社交裂变**：分享战绩激发好友挑战，驱动增长
- **渐进式复杂度**：MVP 先做预设场景 + 预制数字人，验证后迭代
